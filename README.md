Завдання 1

1. Чому аудіо-характеристики винесені в окремий об’єкт audio_features, а не зберігаються плоско? Коли таке вкладення вигідне, а коли створює проблеми?
    Аудіо-характеристики логічно обєднуються в одну групу, так як вони описують конкретні (аудіо) характеристики треку. Логічний об'єкт `audio_features` створює namespace для всієї групи. Це зручно для розуміння документів, адже ми бачитимемо не обєкт з десятками полів, а деревовидну структуру яку легко зрозуміти. Також, це полегшує проекцію у запитах, адже можна одразу всі поля віднесені до одного оюєкту відключити (audio_features: 0). Якщо у майбутньому добавляться нові поля у цей обєкт, це не вплине на проекції верхнього рівня документу. З недоліків я бачу те, що запити, проекції стають довшими через нотацію з крапкою: "audio_features.mode" vs "mode", проте це косметичний недолік.
2. Чому виконавці зберігаються як масив, а не як рядок? Які запити стають простішими?
    У випадку зберігання виконавців як массив ми можемо легко оперувати окремими виконавцями: агрегувати, фільтрувати, сортувати по окремим виконавцям. Усі запити які стосуватимуться окремого виконавця стають простішими. У випадку зберігання одним рядком потрібно буде використовувати regexp, що значно ускладнить і уповільнить запити (адже не можливо буде налаштувати пошукові індекси по окремим артистам).
3. Що таке $out і чим він відрізняється від $merge? Коли використовувати кожен?
    Це оператори aggregation pipeline, обидва використовуються для збереження результатів pipeline у колекці. Основна різниця полягає у тому, що $out повністю видаляє стару колекцію і створює нову. $merge - додає нові документи та оновлює існуючі, документи які не були у pipeline, але є у колекції - не чіпає. Тому $out використовується коли потрібно повністю замінити колекцію не зважаючи, що було у ній раніше (от як у нашому прикладі), побудова якигось "вітрин даних" чи кешів (для прикладу: top N покупок). $merge використовується для інкрементального оновлення існуючих колекцій новими даними.

Завдання 2
1. Для чого використовується інструкція $unwind?
    Це оператор aggregation pipeline, який використовується для розгортання массивів, тобто, у наступний оператор документ буде продубльований стільки ж разів, скільки елементів у вказаному массиві, але замість массива буде відповідне значення з массива. Це необхідно для  роботи з елементами масиву (групування за елментами масиву чи сортування).
2. Чим $stdDevPop відрізняється від $stdDevSamp?
    Обидва оператори використовуються для обчислення стандартного відхилення, проте використовують різні формули для її підрахунку. $stdDevPop - використовується коли наші дані це генеральна сукупність (є всі дані які нас цікавлять, не окрема випадкова вибірка), $stdDevSamp у випадку, якщо у нас лише невелика випадкова вибірка (для прикладу, топ 50/100 треків по жанрам) на основі якої ми хочемо зробити висновок по глобальні тренди. У другому випадку, у формулі в знаменнику N-1 (к-сть елементів).
